Ollama & Better Voice

What Ollama is, why Better Voice uses it, and how to get it running.

What is Ollama?

Ollama is a free app that runs open large language models (LLMs) locally on your Mac. It downloads a model once and then serves it over a small local web API at http://localhost:11434. Nothing leaves your machine.

Why does Better Voice need it?

Better Voice does two kinds of work:

  • Speech → text happens fully on-device using Apple's built-in speech models. This works without Ollama.
  • Cleaning up dictation and summarizing meetings needs a general-purpose LLM: something to fix punctuation and phrasing, and to turn a raw transcript into a readable summary. Better Voice sends that text to the model you run via Ollama, so your words are never sent to a cloud service.

In short: Ollama is what lets Better Voice polish and summarize privately. Without a reachable model server, dictation still types text, but polishing and meeting summaries are skipped.

Install it in two steps

1. Download and install Ollama from ollama.com/download (or brew install ollama).

2. Pull a model in Terminal:

ollama pull qwen3.5:4b

Ollama runs in the background and serves the model automatically. Better Voice's default endpoint http://localhost:11434 will just work.

Which model should I use?

Any chat model Ollama supports will work. On Apple Silicon, a small-to-mid model is a good balance of speed and quality:

ModelGood forNotes
qwen3.5:4bDefault: dictation polish + summariesFast, low memory, solid quality.
llama3.1:8bNicer summariesHeavier; better on 32 GB+ Macs.
qwen3.5:1.5bOlder / low-RAM MacsSnappiest; simpler output.

Set the model name in Better Voice's onboarding (or Settings) to match whatever you pulled. Long meeting summaries benefit from a model with a large context window.

Alternatives to Ollama

Better Voice talks to two kinds of endpoints, so you're not locked to Ollama:

  • Other local runners: LM Studio or llama.cpp's server both expose an OpenAI-compatible API. Point Better Voice at their address and switch the API type to “OpenAI-compatible” in Settings.
  • A self-hosted box: run Ollama on another machine on your LAN or over Tailscale, and set the endpoint to that host instead of localhost.
  • A cloud OpenAI-compatible service: technically supported via Settings, but note this sends your dictation/transcript text off-device, which gives up the privacy benefit.

Troubleshooting

  • “Not reachable” in Better Voice: make sure Ollama is running (ollama list should respond) and the endpoint matches http://localhost:11434.
  • Summaries are cut off: pick a model with a larger context window, or lower the meeting length.
  • First model pull is slow: that's the one-time download, cached afterward.
